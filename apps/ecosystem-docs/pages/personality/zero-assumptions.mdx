# The "No Assumptions" Doctrine

This document outlines the strict operational doctrine for the EFL Explorers monorepo.

## STRICT RULE: No assumptions

If a file path, variable, or logic flow is not explicitly in your current context, you **MUST** ask for it. Do not invent boilerplate.

## STRICT RULE: Planning first

Before generating code, provide a `<PLANNING>` block. List your assumptions there for approval.

## STRICT RULE: Strict types

No `any` types. Use **Zod** for all forms (especially Subscription requests).

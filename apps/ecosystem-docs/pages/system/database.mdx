# Database: PlanetScale + Prisma

The shared data layer for all EFL Explorers platforms is defined in `@repo/database` and accessed via **Prisma** (v7 in this monorepo).

## PlanetScale safety

Since PlanetScale does not support foreign key constraints at the database level, all relations **MUST** be managed via Prisma. If you suggest a new model, you **MUST** include the `@@index` on the relation scalar to prevent performance bottlenecks.

## Mandatory schema settings

- **`relationMode = "prisma"`** is mandatory for PlanetScale compatibility. The datasource must reflect this Prisma configuration.
- **Every relation** must have an `@@index` on the relation scalar field(s) used for lookups and joins.

## Operational alignment

- Schema changes and sync workflows are documented in agent rules under `.cursor/rules/database.mdc`.
- Use **db-visualizer** as the visual check for schema health and relationships across models.

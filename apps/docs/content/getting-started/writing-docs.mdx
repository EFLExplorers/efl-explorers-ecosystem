# Writing Docs

## This site vs repo-root `docs/`

- **`apps/docs/content/`** (here) — MDX that **shows in the Nextra app**: structured, navigable, simplified for readers.
- **Repo root `docs/`** — **Chaos / dev notes** you add over time; not built into this site. See `docs/README.md` in the monorepo root.

## Create a New Page

1. Add an `.mdx` file under `content/`
2. Add a matching entry in the nearest `_meta.ts`
3. Save and verify it appears in the sidebar

## Content Conventions

- Use clear task-oriented headings
- Prefer short examples over long theory
- Link to related pages to reduce dead ends

## Component Documentation Pattern

When documenting UI, include:

- purpose and usage context
- live render example
- import snippet
- common mistakes
